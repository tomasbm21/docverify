Extraction Module
Multilingual Field Extraction
200+ synonym mappings across 5 languages. 10-step extraction pipeline with cascading fallbacks.

10-Step Extraction Pipeline

1. Classify

Regex keyword signals for 5 doc types. Handles spaced-out titles. Confidence: 0.9/0.6.

2. Table KV

Key-value pairs from 2-column metadata tables and wide tables with spacer columns.

3. Text KV

Regex label:value pairs from free-text lines.

4. Confirmation

Pattern-match fields from prose confirmations.

5. Merge

Priority: tables > text > confirmation.

6. Headers

Score header row candidates by recognized fields.

7. Line Items

Structured tables + legacy fixed-width parser.

8. Totals

From TOTAL rows, text patterns, or sum items.

9. Confidence

Extraction confidence score from field coverage.

10. LLM

Optional: Ollama/Anthropic/Gemini for unresolved labels.

Matching Module
Union-Find Shipment Clustering
5-phase matching: exact edges → raw clusters → fuzzy fallback (≥85) → re-collect → build groups.

Algorithm

// Phase 1: Exact-match edges via Union-Find for doc in docs: for field in (bl_no, order_no, container_no, reference): if normalized_ids[field] match → union(i, j) // Phase 2: Collect raw clusters // Phase 3: Fuzzy fallback for singletons (rapidfuzz ≥ 85) // Phase 4: Re-collect after fuzzy merge // Phase 5: Build ShipmentGroups with certainty scores // 1.0 = 2+ exact, 0.7 = 1 match, 0.4 = fuzzy, 0.2 = singleton
