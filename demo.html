Stage Details

A — Ingestion

Reads .docx (paragraphs + tables interleaved), .xlsx (merged cells, metadata rows), .pdf (multi-page). SHA-256 content hash for dedup. Never crashes on bad files.

ingest_file(path) → RawDoc

B — Extraction

10-step extraction: classify → tables > text > confirmation patterns. 200+ multilingual synonym mappings. Legacy fixed-width parser. Optional LLM fallback.

extract_one(raw) → CanonicalDoc

C — Matching

Weighted quick-union with path compression. Exact-match edges → fuzzy fallback (rapidfuzz ≥ 85). Majority-vote consensus identifiers. Deterministic G01..Gnn IDs.

match(docs) → ShipmentGroup[]

D — Verification

7 comparison passes: identifiers (HIGH), totals numeric (HIGH), totals count (HIGH), currency (HIGH), parties (MEDIUM), logistics (LOW), line items (HIGH).

verify(groups, docs) → Verdict[]

E — Reporting

Generates results.json, per-shipment markdown reports, correction-request email drafts, and self-contained dashboard.html with inline data.

report(verdicts, docs) → dict

06 — Extraction Module
Multilingual Field Extraction
200+ synonym mappings across 5 languages. Three-tier metadata extraction: structured tables → free-text KV pairs → confirmation prose.

10-Step Extraction Pipeline

1. Classify

Regex keyword signals for 5 doc types. Handles spaced-out titles. Confidence: 0.9 (header) / 0.6 (body).

2. Table KV

Extract key-value pairs from 2-column metadata tables and wide tables with spacer columns.

3. Text KV

Regex label:value pairs from free-text lines. Supports multiple languages.

4. Confirmation

Pattern-match order_no, reference, bl_no, container_no, shipper/consignee, vessel from prose.

5. Merge Meta

Priority: tables > text > confirmation. Build identifiers, parties, logistics objects.

6. Header Detect

Score header row candidates by recognized fields. Prefer line-item headers over metadata.

7. Line Items

Parse structured tables + legacy fixed-width. Compute amount fallback from price × cartons.

8. Totals

Extract from TOTAL rows, paragraph text, or sum line items. Regex for EN and Arabic patterns.

9. Confidence

Compute extraction_confidence based on field coverage and source quality.

10. LLM Fallback

Optional: send unresolved labels to Ollama/Anthropic/Gemini. Validates against canonical field set.

07 — Matching Module
Union-Find Shipment Clustering
5-phase matching: exact-match edges → raw clusters → fuzzy fallback (≥ 85) → re-collect → build ShipmentGroups with certainty scores.

Algorithm

// Phase 1: Exact-match edges via Union-Find for doc in docs: for field in (bl_no, order_no, container_no, reference): if normalized_ids[field] match → union(doc_i, doc_j) // Phase 2: Collect raw clusters clusters = collect_from_union_find() // Phase 3: Fuzzy fallback for singletons for singleton in singletons: for cluster in clusters: if rapidfuzz.token_set_ratio ≥ 85 → merge // Phase 4: Re-collect after fuzzy merge // Phase 5: Build ShipmentGroups with certainty scores // 1.0 = 2+ exact matches, 0.7 = 1 match // 0.4 = fuzzy, 0.2 = singleton

08 — Verification Module
7-Pass Cross-Reference Engine
Majority-vote outlier detection. Tolerance-aware numeric comparison. Line items matched by lot then by description.

Comparison Passes

HIGH

Identifiers

order_no, bl_no, container_no, reference — exact match after normalization (uppercase, strip separators).

HIGH

Totals Numeric

cartons, net_kg, gross_kg, value — tolerance-aware comparison. Default tolerance: 0.0.

HIGH

Currency

Currency code comparison across all documents in a shipment group.

MEDIUM

Parties

Shipper/consignee normalized string comparison (lowercase, collapse whitespace).

LOW

Logistics

vessel, voyage, pol, pod, ship_date — note: date format differences often cause LOW findings.

HIGH

Line Items (by lot)

Match line items across documents by lot number first. Compare cartons, net_kg, gross_kg, price, amount.

HIGH

Line Items (by desc)

Second pass: match by normalized description if lot match fails. Same numeric field comparison.

INFO

Suspect Collection

Collect doc_ids that are minority outliers. In 2-doc ambiguous case, both marked as suspects.

09 — LLM Module
Multi-Provider LLM Router
Optional LLM fallback for unresolved field labels. Switchable between Ollama (local), Anthropic Claude, and Google Gemini via environment variable.
Ollama (Local)DEFAULT

Local inference via Ollama HTTP API. 120s timeout. Default model: llama3.

LLM_PROVIDER=ollama OLLAMA_BASE_URL=http://localhost:11434 OLLAMA_MODEL=llama3 # Endpoint: /api/generate # Health: /api/tags
Anthropic ClaudeCLOUD

Claude Sonnet via Anthropic SDK. max_tokens=4096. Requires API key.

LLM_PROVIDER=anthropic ANTHROPIC_API_KEY=sk-ant-... # Model: claude-sonnet-4-6-20250514 # System prompt separate from user
Google GeminiCLOUD

Gemini 2.0 Flash via google.generativeai SDK. System prepended to user prompt.

LLM_PROVIDER=gemini GEMINI_API_KEY=AIza... # Model: gemini-2.0-flash # max_output_tokens=4096

Label Resolution Flow

10 — Email Module
IMAP Inbox & Resend Sender
Poll IMAP for shipping document attachments. Send correction-request emails via Resend API.

📥 IMAP Inbox

Polls IMAP server for unread emails with .docx/.xlsx/.pdf attachments. Downloads, converts to RawDoc, feeds into pipeline.

EMAIL_HOST=imap.gmail.com EMAIL_USER=... EMAIL_PASSWORD=... # poll_inbox(host, user, password) # → list of downloaded file paths # → fetch_unread_docs() → RawDoc[]

📤 Resend Sender

Send correction-request emails with findings. Supports single/multiple attachments (base64-encoded).

RESEND_API_KEY=re_... EMAIL_FROM=onboarding@resend.dev # send_email(to, subject, html, text) # send_with_attachment(to, subject, body, path) # send_with_multiple_attachments(to, subject, body, paths)

12 — Feedback Loop
Human-in-the-Loop Learning
Append-only JSONL feedback log → metric computation → parameter tuning → synonym boosts + tolerance adjustments → improved accuracy.

📝 FeedbackTracker

Append-only JSONL log. Each record: shipment_id, field, verdict (confirmed/rejected/corrected), notes, severity, doc_type.

tracker.log( shipment_id="G01", field="identifiers.order_no", verdict="confirmed", notes="Verified against B/L" ) metrics = tracker.compute_metrics(window_days=30)

⚙️ FeedbackTuner

Analyzes feedback to adjust pipeline parameters. Boosts synonyms for error-prone fields, sets numeric tolerances, lowers LLM trigger threshold.

// Auto-computed overrides: // - Fields with >30% error → synonym boost // - Corrected weights → 2% numeric tolerance // - Accuracy < 80% → lower LLM trigger tuner.save_overrides() → tuning_overrides.json

15 — Data Schemas
Pydantic v2 Data Models
FROZEN schema definitions — the source of truth for all data flowing between pipeline stages.

RawDoc

doc_id: str # SHA-256 source_path: str source_format: SourceFormat text: str tables: list[list[list[str]]]

CanonicalDoc

doc_id: str source_path: str doc_type: DocType source_format: SourceFormat identifiers: Identifiers parties: Parties logistics: Logistics line_items: list[LineItem] totals: Totals extraction_confidence: float raw_field_labels: list[str]

ShipmentGroup

group_id: str # G01..Gnn doc_ids: list[str] grouping_key: dict match_certainty: dict[str, float]

Finding

group_id: str field: str # e.g. "identifiers.order_no" doc_a: str # consensus doc_id value_a: str # expected value doc_b: str # deviant doc_id value_b: str # found value severity: Severity # high|medium|low message: str

ShipmentVerdict

group_id: str verdict: str # PASS | FAIL suspect_doc_ids: list[str] findings: list[Finding]

Enums

DocType: bill_of_lading | packing_list | commercial_invoice | proforma_invoice | confirmation | unknown SourceFormat: docx | xlsx | pdf Severity: high | medium | low
